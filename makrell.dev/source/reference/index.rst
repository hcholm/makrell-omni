Reference
=========

This section gathers family-wide reference material. It is intended for readers
who already know roughly what part of the Makrell family they want, and now
need terminology, comparisons, or quick answers.

Use this section when you want:

* a quick comparison between implementations and formats
* family-wide terminology
* short answers to common orientation questions
* compatibility entry points for older top-level pages

For deeper normative detail, use the specifications in the repo under
``specs/``.

What is here
------------

``implementation-matrix``
    A practical comparison of the current implementation tracks and their main
    areas of focus.

``glossary``
    A compact reference for recurring Makrell terminology used across the site.

``faq``
    Short answers to common questions about the family structure,
    implementations, and current project shape.

How to use this section
-----------------------

A simple way to use the reference pages is:

#. check the implementation matrix if you need orientation
#. use the glossary if a family term is unclear
#. use the FAQ for short project-shape questions
