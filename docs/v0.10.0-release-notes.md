# Makrell v0.10.0 Release Notes

## Summary

`v0.10.0` is the first release intended to present Makrell as a coherent
family rather than a loose set of experiments.

The theme of this release is:

**consolidation, packaging, and developer experience**

This release does not mean the family is final or `1.0`-stable. It does mean:

- the main implementation tracks are more aligned
- the docs and site tell a clearer current story
- editor/tooling work is being treated as part of the product
- the family has a more credible shared language surface in day-to-day use

## Highlights

### Shared async/await baseline

Makrell now has a shared core async surface across the three main language
tracks:

- `{async fun ...}`
- `{await expr}`

This is now part of the family story rather than a track-specific extra.

### Stronger compile-time parity

Compile-time Makrell has been treated as a release-level concern in this cycle.
In particular:

- MakrellPy continues to provide the strongest compile-time reference
- MakrellTS now has a clearer public macro story and a stronger browser-facing
  tooling path
- Makrell# has gained meaningful compile-time parity improvements, including:
  - shared async/await support
  - compile-time `match`, `~=`, and `!~=`
  - better public showcase macros

The long-term goal remains broader reuse of the same parser/compiler path for
meta and runtime code.

### MRTD is now a real family format

MRTD has moved from draft idea toward family-level reality:

- current draft spec written down
- implementation presence across `.NET`, MakrellTS, and MakrellPy
- typed read/write helpers across the three tracks
- first profile experiment through `extended-scalars`
- documentation and site presence on `makrell.dev`

### Shared editor assets

The language/editor surface now has an explicit shared asset base:

- language metadata
- syntax grammar
- snippets
- language configuration

These assets are now shared between `vscode-makrell` and MakrellTS-side browser
tooling work, rather than being left to drift independently.

### Playground direction on `makrell.dev`

The MakrellTS playground has been raised from a loose idea to a documented
product track for `v0.10.0`:

- dedicated `makrell.dev/playground/` section
- product-surface framing, flows, states, onboarding, and launch notes
- browser host work connected to real MakrellTS/runtime assets
- explicit plan to share editor assets with `vscode-makrell`

## By implementation

### MakrellPy

MakrellPy remains the strongest semantic reference track.

Notable `v0.10.0` material:

- public macro showcase around `pipe`, `rpn`, and `lisp`
- clearer async/await documentation and examples
- continued role as the benchmark for compile-time ergonomics

### MakrellTS

MakrellTS has become much more central to the `v0.10.0` story.

Notable `v0.10.0` material:

- baseline `{async fun ...}` / `{await ...}` support
- stronger package/export structure
- `.mrts` as the TypeScript-side source extension
- exported shared editor assets
- exported playground example manifest
- browser-facing tooling and playground-host groundwork

### Makrell#

Makrell# has moved forward substantially in this cycle.

Notable `v0.10.0` material:

- baseline `{async fun ...}` / `{await ...}` support
- better generic CLR interop ergonomics, including common inferred generic
  static calls
- stronger public showcase examples
- compile-time pattern matching in `meta` and macro code
- clearer docs for interop, macros, async, and tooling

## Tooling and docs

### `makrell.dev`

The site now better reflects the family as it exists today:

- clearer family positioning
- stronger implementation sections
- MRTD section
- playground section
- improved navigation and cross-linking

### `vscode-makrell`

The extension has been moved toward a more honest family-aware shape:

- broader file support
- refreshed snippets and grammar work
- shared asset model for future browser/editor reuse
- TS-first direction for future evolution

## What this release does not mean

`v0.10.0` does **not** mean:

- full cross-track parity
- finalised language design
- complete packaging/publishing polish in every track
- a finished browser playground
- a finished meta/runtime architecture story, especially for Makrell#

This is a consolidation release, not an endpoint.

## Known rough edges

The main remaining rough edges after `v0.10.0` are expected to include:

- broader compile-time/runtime reuse, especially in Makrell#
- deeper packaging/publishing polish
- more complete `vscode-makrell` verification and release flow
- broader parity coverage and fixture work
- fuller public showcase curation across all tracks
- finishing the MakrellTS playground as a real browser product surface

## Suggested release message

Makrell `v0.10.0` is the first consolidation release for the Makrell family:
better aligned language tracks, a shared async baseline, MRTD across the
family, stronger compile-time parity, shared editor assets, and a clearer path
from docs to tools to browser experience.
